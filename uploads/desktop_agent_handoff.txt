============================================================
READY PROMPT TO PASTE INTO NEW DESKTOP AGENT
============================================================
Продолжи программу прошивок с GitHub (kotlin-web-site, PR #1 уже влит в master).

Контекст handoff: uploads/desktop_agent_handoff.txt

Сделай на ЭТОМ компьютере:
1. git pull origin master
2. Найди локальную папку прошивок (Яндекс.Диск / РЕЗЕРВ / firmware_base)
3. Прогони цепочку:
   python scripts/firmware_catalog_ingest.py --source "<LOCAL_FIRMWARE_FOLDER>" --out-dir uploads/yadisk_ingest_run
   python scripts/ident_docs_ingest.py --source "<LOCAL_FIRMWARE_FOLDER>" --out-dir uploads/ident_run --name-mode strict --firmware-index-csv uploads/yadisk_ingest_run/firmware_catalog_index.csv
   python scripts/stage_core_builder.py --index-csv uploads/yadisk_ingest_run/firmware_catalog_index.csv --out-dir uploads/stage_core_run --ident-priority-csv uploads/ident_run/ident_brand_priority_core_brands.csv
   python scripts/family_manifest_builder.py --firmware-index-csv uploads/yadisk_ingest_run/firmware_catalog_index.csv --ident-priority-csv uploads/ident_run/ident_brand_priority_core_brands.csv --stage-families-csv uploads/stage_core_run/stage_core_families.csv --out-dir uploads/family_manifest_run
   python scripts/safety_precheck_builder.py (по uploads/safety_precheck_usage.txt, на свежий manifest)
4. Покажи пути к результатам и краткий summary.
5. Правила каталога — системные на весь каталог брендов, без JAC/Haval/Chery-only хаков.
6. BIN не модифицировать.

Если локальную папку прошивок не найдёшь — остановись и спроси точный путь.
